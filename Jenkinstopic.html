01 — Core Fundamentals
01 What is Jenkins & How It Works Interview
  • CI/CD orchestration server — open-source, Java-based
  • Master/Agent (Controller/Agent) architecture
  • Polling SCM vs Webhooks vs manual triggers
  • Jenkins workspace, executor threads, build queue
  • How Jenkins differs from GitHub Actions / Azure DevOps
02 Freestyle Jobs vs Pipeline Jobs Interview
  • Freestyle: UI-driven, no code, limited reuse
  • Pipeline: code-as-config via Jenkinsfile
  • Declarative Pipeline vs Scripted Pipeline (Groovy DSL)
  • When to use each — interview favourite question
  • Multi-branch Pipeline & Organization Folders
03 Jenkinsfile Structure & Syntax Interview
  • pipeline {} → agent → stages → stage → steps
  • environment {} block for env vars
  • post { always/success/failure/unstable }
  • parameters {} — string, boolean, choice
  • when {} conditional execution
  • options {} — timeout, retry, skipDefaultCheckout
02 — Pipeline Architecture
04 Agents & Executors Technical
  • agent any, agent none, agent { label '...' }
  • Docker agent — agent { docker { image '...' } }
  • Kubernetes agent (pod templates)
  • SSH agents, JNLP agents, inbound vs outbound
  • Executor count tuning, offline agents, node labels
05 Parallel & Matrix Execution Technical
  • parallel { stage('A'){} stage('B'){} }
  • failFast true — abort all branches on failure
  • Matrix axes — OS × JDK cross-testing
  • Parallel test execution for faster feedback loops
  • Stash / unstash to pass artifacts between stages
06 Shared Libraries Technical
  • Structure: vars/, src/, resources/
  • Global vs folder-scoped libraries
  • @Library('my-lib') _ import syntax
  • Writing reusable vars/deploy.groovy steps
  • Version pinning of shared libraries
07 Triggers & Build Strategies Technical
  • GitHub/GitLab Webhooks — GenericTrigger plugin
  • Poll SCM cron syntax: H/5 * * * *
  • Upstream/downstream job chaining (build job:)
  • Periodic build triggers, parameterized triggers
  • Build on PR / branch filter strategy
03 — Security & Credentials
08 Credentials Management Interview
  • Jenkins Credential Store — types: Username/Password, SSH key, Secret text, Certificate, Secret file
  • Scopes: Global vs System vs Folder
  • withCredentials([]) binding in pipeline
  • Never echo secrets — Jenkins masks credentials in logs
  • Integrating with HashiCorp Vault / Azure Key Vault
09 Role-Based Access Control (RBAC) Interview
  • Matrix Authorization Strategy plugin
  • Role Strategy plugin — roles, users, groups
  • Project-level permissions vs global permissions
  • Integration with LDAP / Active Directory (your strength!)
  • SSO via SAML / OAuth2 plugins

05 — Advanced Topics
12 Jenkins as Code — JCasC Advanced
  • Jenkins Configuration as Code plugin (JCasC)
  • Define entire Jenkins config via YAML
  • Manage jobs via Job DSL plugin (Groovy)
  • Seed jobs — auto-generate pipelines from repo
  • GitOps approach: Jenkins config in version control
13 Jenkins on Kubernetes Advanced
  • Deploy Jenkins via Helm chart
  • Dynamic agents using Kubernetes plugin (pod templates)
  • Persistent volume for Jenkins home
  • Scaling — ephemeral pods per build
  • Jenkins X — cloud-native CI/CD on K8s
14 High Availability & Scaling Advanced
  • Jenkins HA setup — CloudBees HA (active/passive)
  • Distributed builds across multiple agents
  • Load balancing build executors
  • Backup of $JENKINS_HOME — ThinBackup plugin
  • Disaster recovery — restoring from backup
15 Pipeline Optimization & Best Practices Advanced
  • Minimize checkout overhead — shallow clone
  • Cache dependencies between builds (node_modules, .m2)
  • Use options { skipStagesAfterUnstable() }
  • Keep Jenkinsfile lean — move logic to shared libs
  • Build once, deploy many — promote artifacts
  • Immutable artifacts — never rebuild for each env
06 — Day-to-Day Business Support
🔧 Build Management
  • Monitor running / queued builds
  • Abort stuck builds, free executors
  • Replay failed pipeline from last run
  • Trigger parameterized builds manually
  • Archive / discard old build artifacts
🔍 Troubleshooting Failures
  • Read Console Output — identify root cause
  • SSH into agent to debug workspace
  • Fix agent offline — reconnect / restart
  • Resolve plugin version conflicts
  • Diagnose OOM kills in build containers
🚀 Deployment Support
  • Promote builds from Dev → QA → Prod
  • Rollback by redeploying a previous build
  • Manage approval gates (input step)
  • Deploy hotfixes via pipeline parameters
  • Blue/Green deploy orchestration via scripts
🔒 Security & Compliance
  • Rotate credentials in Jenkins Credential Store
  • Audit who triggered which build (Audit Trail plugin)
  • Enforce SonarQube quality gates — block bad code
  • Restrict prod deployments to specific users
  • Plugin security advisories — patch promptly
📊 Monitoring & Alerts
  • Slack / Email alerts on build failure
  • Monitor Jenkins metrics via Prometheus plugin
  • Build trend charts — identify flaky tests
  • Disk usage alerts for workspace bloat
  • Jenkins log rotation & cleanup jobs
🛠️ Admin & Maintenance
  • Plugin updates — test in staging first
  • Jenkins version upgrades (LTS track)
  • Add / configure new build agents
  • Manage Jenkins home disk space
  • Periodic $JENKINS_HOME backups
07 — Suggested Learning Roadmap
WEEK 1
Foundations — Install Jenkins, understand architecture, create Freestyle & Declarative pipeline jobs, connect to GitHub via webhook
WEEK 2
Pipelines — Master Jenkinsfile syntax, parallel stages, stash/unstash, environment blocks, post conditions, build parameters
WEEK 3
Agents & Docker — Set up Linux agent via SSH, Docker-in-Docker pipeline, build & push image to DockerHub/ACR, deploy to Azure VM (continue your project!)
WEEK 4
Security & Quality — Credentials management, RBAC, LDAP/AD integration, SonarQube quality gates, audit trail setup
WEEK 5–6
Advanced — Shared Libraries, JCasC, Kubernetes agent plugin, pipeline optimization, HA considerations, Prometheus monitoring
